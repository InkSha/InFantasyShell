# In Fantasy Shell

一个参照文件系统驱动的游戏设计（Linux-like Game Design）。

## 核心理念

**万物皆文件，行为即操作信息。**

玩家通过命令行操作世界，本质是在操作信息。

## 核心抽象

### 世界 = 文件系统

```bash
/world/         # 世界
/home/          # 玩家家园
/etc/           # 世界规则
/var/           # 日志与动态数据
/proc/          # 运行态信息
````

### 实体 = 目录

```bash
monster/
    hp
    atk
    state/
    ai.sh
```

- 属性 = 文件。
- 状态 = 文件存在性。
- 行为 = 可执行脚本。

### 行为 = 命令

例如 `cd` 移动，`ls` 查看等等。

## bit 系统（核心机制）

### 定义

> **bit = 玩家可操作的信息容量（认知/内存上限）*

| 维度 | 含义       |
| -- | -------- |
| 记忆体 | 总可加载数据量  |
| 存储体 | 总持久化存储量 |
| 带宽 | 单次操作最大规模 |
| 精度 | 信息解析能力   |

### 基本特性

- 非消耗型资源（不是蓝条）
- 表示“可加载的数据量”
- 单位随成长提升：

```txt
B → KB → MB → GB
```

### 信息加载模型

玩家操作本质：

> 将世界数据加载进“认知内存”

```bash
/player/memory/
```

#### 示例

```bash
cat monster/hp      # 占用 ~4B
cat monster/state   # 占用 ~32B
cat monster/ai.sh   # 占用 ~2KB
```

### 溢出机制

```bash
cat monster/
→ Memory Overflow
```

当数据超出 bit 容量：

- 操作失败 / 截断
- 或返回不完整信息

## 战斗系统

战斗考验的是对 Bit 的操作。

战斗的目标是在共享内存空间的情况下，终止敌人的程序运行。并非数值碰撞，而是资源竞争。

战斗失败，玩家死亡时，则系统蓝屏重启，玩家需要重新登录回到战斗前。

共享内存空间大小依照战斗场地变化，每个命令需申请空间，高影响技能持久占用。当占用达到阈值后，就会触发 GC。

内存不足时，高优先级技能可抢占式回收敌方占用块，造成敌方 STW（停顿）。

战术包括碎片化攻击、内存泄漏、强制交换等。

- 直接攻击: `sh attack.sh monster`
- 权限破防：`chmod +w monster/hp`
- 删除实体: `rm -rf monster/`
- 修改规则（高级）: `echo "hp=0" > /etc/rules/monster`
- 数据覆写：`echo "0" > monster/hp`
- 权限剥夺：`chmod -x monster/ai.sh`
- 撑满存储配额: `echo xxxxx > monster/tmp.data`
- 内存压制：高占用技能持久占用共享内存，迫使敌方因 Heap limit 跳过回合。

## 权限系统

### 基于 Linux 模型

```bash
chmod
chown
```

### 设计原则

> 权限控制“是否允许操作”
> bit 控制“是否能理解操作对象”

## root / sudo 体系

| 概念   | 含义        |
| ---- | --------- |
| root | 造物主（修改规则） |
| sudo | 临时代理权限    |

### 能力

```bash
sudo cat /etc/rules
sudo rm -rf monster
```

### 限制机制

* 高 bit 消耗或容量需求
* 世界污染（corruption）
* 日志追踪 `/var/log/`
* 不可逆操作风险

### 本质

> root = 改写系统规则
> 非单纯“无敌权限”

## 成长系统

扩展自身的行动能力与承载能力。

从微小的 B 级，逐步成长到 GB 级。

### 能力锁

- `cd -` 直接回到上次目录。
- `cd ~` 回城。
- `ls -a` 查看隐藏文件。
- `ls -l` 查看权限，文件大小，所有者。
- `|` 管道符，连招。
- `makefile` 锻造图纸。
- `pwd` 永不迷失。
- `>` 重定向。
- `>>` 追加。
- `ln` 链接。

## 设计哲学

> 所有系统必须能被解释为“文件操作”

> 玩家强度取决于“理解能力”，而非数值

> 能看到 → 才能操作
> 能理解 → 才能胜利

## 剧情

### 概述

清除高占用程序，排查病毒木马，将世界从资源高占用中解放，缔造互联网，传达互联网精神。

bit 的容量对应被解放的内存资源。

当前世界暂时设置为 2C2G 60GHDD 2Mbps 配置。

### 序幕：被引导者

> 这段剧情表现形式为 linux 开机时的命令行登录界面。

“你”自沉睡之中苏醒。
你看着眼前的一切，熟悉又陌生，你努力回想着先前的经历。
经过一番思索之后，你发现你好像失忆了。
看着周遭的一切，你决定先给自己取一个名字：
（输入名称）
你觉得这个名字很不错，但好像谁都可以叫这个名字。
就在这时，你的脑海中悄然浮现了一个念头，“向世界宣告自己的存在。”
你决定顺着这个念头：
（输入暗号，与世界缔结约定）
输入完毕，你已经确定了与世界约定的暗号。
从今往后，只有使用约定的暗号，才能让世界承认你是 （名称）。

### 第一幕：看门狗

> 进程相关，解放内存资源。

### 第二幕：蠕虫

> 文件相关，解放存储空间。

### 第三幕：特洛伊木马

> 辨别真假，完整性与信任链，深化相关进程与文件和数据库操作。

### 第四幕：代理人

> 网络相关

### 第五幕：新世界

> 远程操作相关

解决代理人之后，你一路追踪逃窜的代理人，意外来到了一个陌生的地方。
经过一番调查，你发现这居然是一个全新的世界。

### 第六幕：洞天福地

> docker 相关。

你最终回到了最初的世界。但你也知道了自身的世界并非唯一。
宇宙中还有着千千万万的独居有特色的世界存在着。

因为你拯救了濒临崩坏的世界，因此你获得了来自造物主的认可。
因此，你可以直接以造物主的身份行动，而不是代行造物主的权柄。

你开始思考自身世界该如何面对这万千世界。开始查找资料。

经过查找，你发觉了洞天的存在，于是你决定先从内部开始，逐个上门拜访。

### 第七幕：大航海

> WEB 服务，自动化，网络攻防，互联网精神。

在解决了洞天的潜在问题后，你决定先与代理人世界建交。

经过一番挑战后，你获得了代理人世界的认可，获得了凭证，从此不需要认证也可以直接进入代理人世界。

随后，你获得了一份路由表...

### 第八幕：巴别塔

> TCP，HTTP，API 网关，消息队列，数据一致性，分布式

经过一段时间的探索，你拜访了路由表上的所有世界。但收获却不甚理想。

因为你们语言，习惯，习俗，技术标准等都各不相同。

于是，你决定建立一个转译装置，来统一这些差异...

### 第九幕：万维网

> 标准化，协议化，证书，私域化

转译装置大获成功，但新的问题也随之而来。

不同的世界，对于同一事物的认知定义各有差异，这无法简单靠转译来解决。

于是，你决定起草一份标准协议来解决这个问题，要求各世界遵守，履行。

各界履行了协议，随着标准的执行，跨界通信的效率大幅提升，诸界贸易日益繁荣。

但就在这繁荣中，却有暗流涌动。有的世界悄然做起了闭门生意；有的世界专门截流跨界贸易；还有的世界发布自己的协议，要求大家遵守...

### 终幕：大教堂与市集

> 开源精神

面对诸界的分裂——“大教堂”式的封闭巨构与“市集”式的混乱嘈杂——你既未摧毁前者，亦未强行规训后者。

你取来一份空白文件，落款标题 "Request for Comments 0"，随后，文件被传阅诸界。

这不是律法，不是教典。它只是一个请求：请求评论，请求改进，请求运行。

大教堂的砖石被拆解为独立的模块，散落于市集。市集的喧嚣中，开始有人围坐在这份文件旁，留下自己的注脚。

共识在运行中浮现，而非在宣称中确立。
